# GED Viewer Plugin для Obsidian

Плагин позволяет интегрировать генеалогические данные из GEDCOM-файлов прямо в заметки Obsidian. Отображайте персональную информацию, семейные отношения и события жизни в структурированном формате.

## Возможности

- Загрузка и парсинг GEDCOM (.ged) файлов из вашего хранилища
- Отображение персональной информации в таблицах с помощью блоков `gedcom-person`
- Показ семейных отношений с блоками `gedcom-family`
- Список событий жизни с блоками `gedcom-events`
- Сравнение несколькихperson в таблицах сравнения
- Доступ ко всем person из GEDCOM-файлов через поисковую команду

## Установка

### Вариант 1: BRAT (Beta Reviewers)
1. Установите плагин [BRAT](https://github.com/TfTHacker/obsidian42-brat)
2. В настройках BRAT нажмите "Add Beta plugin"
3. Введите URL репозитория: `https://github.com/geneology-code/obsidian-ged-viewer`
4. Включите "GED Viewer" в сообществе плагинов Obsidian

### Вариант 2: Ручная установка
1. Скачайте файлы плагина в директорию `.obsidian/plugins/` вашего хранилища
2. Включите плагин в настройках Obsidian
3. Настройте путь к вашему GEDCOM-файлу(ам) в настройках плагина

## Настройка

В настройках плагина укажите путь к GEDCOM-файлу(ам):

- Для одного файла: `path/to/my-family.ged`
- Для нескольких файлов: `path/to/file1.ged, path/to/file2.ged` или `path/to/file1.ged path/to/file2.ged`
- Также можно назначить алиасы: `path/to/file1.ged::My Family Tree`

Включите "Reload on startup" для автоматической загрузки GEDCOM данных при запуске Obsidian.

### Настройки

| Настройка | Описание | По умолчанию |
|----------|----------|--------------|
| GEDCOM file path | Путь к GEDCOM-файлу(ам) | (пусто) |
| Максимальный предположительный период жизни (лет) | Максимальный возраст для person без события смерти. Ограничивает период на временной шкале, чтобы не показывать person как живых бесконечно. | 100 |

**Почему настройка продолжительности жизни важна:**

Когда у person есть дата рождения, но нет даты смерти, плагин ограничивает их временную шкалу максимальной продолжительностью жизни. Например:
- Родился в 1920, нет даты смерти, макс. продолжительность 100 → временная шкала показывает `1920–2020`
- Родился в 2020, нет даты смерти, макс. продолжительность 100 → временная шкала показывает `2020–2026` (текущий год)
- Родился в 1920, умер в 1990 → временная шкала показывает `1920–1990` (фактические даты)

## Использование

### Команды

- **Show all persons from GEDCOM**: Открывает поисковую модаль со всеми person из загруженных GEDCOM файлов
- **Reload GEDCOM data**: Вручную перезагружает GEDCOM данные из настроенных файлов

### GEDCOM Search View

Нажмите на иконку поиска в левой ленте, чтобы открыть панель поиска GEDCOM в правой боковой панели.

**Возможности:**
- **Поиск без учёта регистра** по умолчанию
- **Поддержка regex** - установите флажок "Regex" для использования регулярных выражений
- **Поиск по всем полям** - имя, ID, даты рождения/смерти и места
- **Двойной клик для копирования** - дважды кликните по person, чтобы скопировать их `@ID@`
- **Сортируемые колонки** - кликните по заголовкам колонок (ID, Name, Life Dates) для сортировки
- **Нормализованный формат дат** - даты отображаются в читаемом формате `DD.MM.YYYY`
- **Подсказки при наведении** - наведите на даты жизни для просмотра полной информации о рождении/смерти с местами

**Как использовать:**
1. Нажмите на иконку поиска (🔍) в левой ленте
2. Панель поиска откроется в правой боковой панели
3. Введите поисковый запрос
4. Переключите "Regex" для поиска регулярных выражений
5. Кликните по заголовкам колонок для сортировки результатов
6. Наведите на даты для просмотра детальной информации (даты рождения/смерти и места)
7. Дважды кликните по person для копирования их ID

### Блоки кода

#### Персональная информация (`gedcom-person`)

Отображение персональных деталей для одного или нескольких person:

````markdown
```gedcom-person
@I1@
```
````

Для сравнения нескольких person:

````markdown
```gedcom-person
@I1@ @I2@
```
````

#### Семейные отношения (`gedcom-family`)

Показ информации о семье для одного или нескольких person:

````markdown
```gedcom-family
@I1@
```
````

Для сравнения нескольких person:

````markdown
```gedcom-family
@I1@ @I2@
```
````

#### События жизни (`gedcom-events`)

Отображение событий жизни для одного или нескольких person:

````markdown
```gedcom-events
@I1@
```
````

Для сравнения нескольких person:

````markdown
```gedcom-events
@I1@ @I2@
```
````

#### Интеграция с Timeline (`ged-chronos`)

Создание интерактивных временных шкал с помощью плагина [Chronos Timeline](https://github.com/obsidian-plugin-chronos/obsidian-plugin-chronos). Этот блок разворачивает ссылки на person/семью из GEDCOM в события временной шкалы.

**Синтаксис:**

- `gci: @IX@` — развернуть события для индивида X
- `gcf: @FX@` — развернуть события для семьи X
- Прямые события временной шкалы в формате Chronos

````markdown
```ged-chronos
gci: @I1@
gcf: @F1@
- [1900~1970] French Revolution
```
````

**Как это работает:**

1. Блок `ged-chronos` парсит директивы GEDCOM (`gci:`/`gcf:`)
2. Извлекает рождения, смерти, браки и другие события жизни из GEDCOM данных
3. Напрямую вызывает внутренний метод `_renderChronosPlugin плагина Chronos Timeline`
4. Временная шкала рендерится со всеми возможностями Chronos (зум, панорамирование, переключение ширины)

**Требования:**

- Плагин Chronos Timeline должен быть установлен и включен
- GEDCOM данные должны быть загружены (настроены в настройках плагина)
