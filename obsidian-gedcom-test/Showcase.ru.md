# GEDCOM Genealogy Plugin — Showcase

> Плагин для отображения генеалогических данных из **GEDCOM-файлов** прямо в Obsidian. Загрузите `.ged` файл — и получите карточки персон, таблицы сравнения, интерактивные диаграммы предков/потомков, временные шкалы и даже возможность писать собственные скрипты на JavaScript для доступа к данным.

## Описание плагина

### Что умеет

| Возможность              | Блок                             | Описание                                                                                                                       |
| ------------------------ | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Карточка персоны         | `ged-person`                     | Ключ-значение: имя, даты, места                                                                                                |
| Полная карточка          | `ged-person-full`                | Карточка + дерево родственников                                                                                                |
| Сравнение персон         | `ged-person-compare`, `ged-comp` | Таблица с полями нескольких персон                                                                                             |
| События по датам         | `ged-person-events`              | Хронологическая таблица всех событий                                                                                           |
| Родственники             | `ged-relatives`                  | Родители, супруги, дети, сиблинги                                                                                              |
| Диаграмма предков        | `ged-diagram-ancestors`          | Интерактивное дерево восходящих предков                                                                                        |
| Диаграмма потомков       | `ged-diagram-descendants`        | Интерактивное дерево нисходящих потомков                                                                                       |
| Песочные часы            | `ged-diagram-hourglass`          | Предки + потомки одновременно                                                                                                  |
| Все родственники         | `ged-diagram-relatives`          | Полная сеть связей                                                                                                             |
| Временная шкала          | `ged-chronos`                    | Интеграция с плагином Chronos, позволяет использовать как стандартные события chronos так и "вытаскивать" события персон\семей |
| Пользовательские скрипты | `ged-js`                         | JavaScript с доступом к GEDCOM-данным                                                                                          |

### Формат GEDCOM ID

Во всех блоках используются идентификаторы GEDCOM в **стандартном формате** с обрамлением символами `@`:

```
@I37@    — идентификатор персоны
@F4@     — идентификатор семьи
```

Чтобы найти персону или семью, найдите в боковой панели иконку "[[#GEDCOM Search View|Открыть поиск GEDCOM]]", двойной клик по результату поиска — скопирует его идентификатор.

### Что если данные не загружены

Если GEDCOM-файл не загружен (не указан путь или ошибка парсинга), блоки покажут сообщение:
- Карточки: «Данные GEDCOM не загружены» или «Файл GEDCOM не найден»
- Диаграммы: «Нет данных для диаграммы»
- Блоки с событиями: пустая таблица или сообщение об ошибке

Убедитесь что путь к файлу указан корректно в [[#Настройки плагина]].

## Настройки плагина

Настройки находятся: **Settings → Community plugins → GEDCOM Genealogy → ⚙️**

### Путь к файлу GEDCOM

**Ключ:** `gedcomFilePath`

Путь к вашему `.ged` файлу **относительно корня vault**.

```
Примеры:
royal92.ged                    — файл в корне vault
data/family.ged                — файл в папке data/
ancestors/romanov/royal92.ged    — вложенная структура
```

> **Важно:** файл должен быть в vault (папке Obsidian), не за его пределами. Плагин читает файл через Obsidian Vault API.

После изменения пути **перезагрузите Obsidian** для применения.

---

### Максимальный возраст (без смерти)

**Ключ:** `maxLifespanYears`  
**По умолчанию:** `100`

Максимальный предполагаемый возраст для персон **без события смерти**. Используется для расчёта дат в диаграммах и временных шкалах.

```
Пример:
- Если человек родился в 1800 и нет даты смерти:
  - При значении 100 → предполагаемая смерть: 1900
  - При значении 120 → предполагаемая смерть: 1920
- Если человек умер в 1900 и нет даты рождения:
  - При значении 100 → предполагаемая дата рождения: 1800
  - При значении 80 → предполагаемая дата рождения: 1820
```

> Увеличьте значение если в вашей базе встречаются долгожители.

---

### Количество поколений по умолчанию

**Ключ:** `defaultDiagramGenerations`  
**По умолчанию:** `3`

Количество поколений по умолчанию для диаграмм, если не указан параметр `LVL:N` в блоке кода.

```
Примеры:
- LVL:1 → 1 поколение (только родители/дети)
- LVL:2 → 2 поколения (бабушки/дедушки, внуки)
- LVL:3 → 3 поколения (по умолчанию)
- Без LVL → берётся это значение из настроек
```

> **Рекомендация:** для больших баз данных оставьте 2-3. Для `ged-diagram-hourglass` и `ged-diagram-relatives` не ставьте больше 2 — см. предупреждения ниже.

---

### Отладочные логи

**Ключ:** `enableDebugLogging`  
**По умолчанию:** `Выключено`

Включает детальные отладочные логи для Topola-диаграмм и других функций плагина.

- Выводит информацию о размерах SVG, позициях элементов, масштабе
- Полезно при отладке проблем с отображением диаграмм
- **Требует перезагрузки Obsidian** после изменения

> Включайте только при наличии проблем. Для обычного использования — оставьте выключенным.

---

### Блоки ged-js

**Ключ:** `enableGedJS`  
**По умолчанию:** `Выключено`

Разрешает выполнение **пользовательского JavaScript** в блоках `ged-js`.

> ⚠️ **Предупреждение о безопасности:** код выполняется с **полным доступом** к Node.js API (`require`, `process`, `fs`). Используйте **только с доверенным кодом**, который вы написали или проверили сами.

После включения:
- Блоки `ged-js` начинают выполняться
- Объект `ged` даёт доступ к `ged.service`, `ged.app`, `ged.renderMarkdown()` и др.
- При отключении — блоки показывают сообщение «disabled»

Подробная документация по `ged-js` — в разделе [[#Скрипты (ged-js)]] ниже.

## GEDCOM Search View

Интерактивный поиск по данным GEDCOM в правой боковой панели.

### Как открыть

- **Иконка в ribbon** (левая панель) — 🔍 «GEDCOM Search»
- **Команда** (`Ctrl/Cmd + P`) → «GEDCOM Search: Open GEDCOM Search»

### Возможности

| Функция | Описание |
|---------|----------|
| **Вкладка «Персоны»** | Поиск по имени, ID, датам жизни |
| **Вкладка «Семьи»** | Поиск по ID семьи, датам брака, кол-ву детей |
| **Поиск** | Текстовый поиск + поддержка Regex (галочка) |
| **Сортировка** | Клик по заголовку колонки — сортировка A-Z / Z-A |
| **Вставка блока** | Кнопка `+` — вставляет код блок для персоны/семьи в активную заметку |
| **Копирование ID** | Клик по ID — копирует в буфер обмена |

### Вставка блоков из поиска

Нажмите `+` в строке персоны или семьи → выберите тип блока → код вставится в текущую заметку на позицию курсора.

Доступные типы: `ged-person`, `ged-person-full`, `ged-person-compare`, `ged-person-events`, `ged-relatives`, `ged-chronos`, `ged-diagram-ancestors`, `ged-diagram-descendants`, `ged-diagram-hourglass`, `ged-diagram-relatives`, `ged-js`.

Для семей: `ged-chronos` (через `gcf: @F@`).

Все команды вызываются через **Ctrl+P** (или **Cmd+P** на Mac) → ввод названия.

### «Выберите человека»

Открывает модальное окно со списком **всех персон и семей** из загруженного GEDCOM. Позволяет быстро найти нужную персону, скопировать её ID или сразу вставить блок в заметку.

## Карточки персон

### ged-person

> Выводит информацию в формате `*ключ*: значение` о **первом** человеке из списка ID. Даже если указано несколько `@ID@`, покажет только первого.

**Когда использовать:** быстрая справка по одному человеку. Если нужна полная картина с родственниками — используйте `ged-person-full`.

**Синтаксис:** `@ID@ [@ID@ ...]` — но используется только первый ID.

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-person» (Ctrl+P)*

#### Николай II

```ged-person
@I37@
```

#### Александр III — первый из списка

```ged-person
@I40@ @I41@ @I37@
```

> Несмотря на три ID, показан только @I40@ (Александр III).

### ged-person-compare (ged-comp)

> Всегда выводит **таблицу сравнения**, независимо от количества человек. Алиас `ged-comp` — короткая версия.

**Когда использовать:** сравнение нескольких персон (супруги, поколения, сиблинги). `ged-comp` — удобная короткая форма.

**Синтаксис:** `@ID@ [@ID@ ...]` — все указанные персоны.

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-person-compare» (Ctrl+P)*

#### Сравнение: Николай II, Александра Фёдоровна, Александр III

```ged-comp
@I37@ @I39@ @I40@
```

#### Полная таблица: четыре поколения Romanov

```ged-person-compare
@I42@ @I44@ @I40@ @I37@
```

> Nicholas I → Alexander II → Alexander III → Nicholas II — четыре поколения царей.

#### Пара: супруги

```ged-comp
@I37@ @I39@
```

### ged-person-events

> Таблица **событий по датам** — рождение, браки, смерть, дополнительные события. Хронологическая сортировка.

**Когда использовать:** просмотр жизненного пути персоны или сравнение ключевых дат нескольких человек. Поддерживаются **все типы событий** из GEDCOM — стандартные (BIRT, DEAT, MARR) и пользовательские (Occupation, Residence, Education и т.д.).

**Синтаксис:** `@ID@ [@ID@ ...]` — все указанные персоны.

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-person-events» (Ctrl+P)*

#### Николай II — события жизни

```ged-person-events
@I37@
```

> Включает: рождение, коронацию (EVENT с типом "Coronation" — без перевода), эмиграцию, смерть.

#### Четыре поколения — сравнение событий

```ged-person-events
@I42@ @I44@ @I40@ @I37@
```

> Показывает как сдвигаются даты жизни каждого поколения.

#### Семья Nicholas II — все события

```ged-person-events
@I37@ @I39@ @I46@ @I47@ @I48@ @I49@ @I50@
```

### ged-person-full

> Выводит **всю информацию** о персоне: сначала key:value детали (как `ged-person`), затем дерево родственников в формате markdown (как `ged-relatives`).

**Когда использовать:** полная справка по персоне с обзором семьи. От `ged-person` отличается наличием дерева родственников. От `ged-relatives` — наличием key:value деталей сверху.

**Синтаксис:** `@ID@ [@ID@ ...]` — используется только первый ID.

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-person-full» (Ctrl+P)*

#### Николай II — полная информация

```ged-person-full
@I37@
```

> Ключи → родители → супруга → 5 детей.

#### Александра Фёдоровна — полная информация

```ged-person-full
@I39@
```

> Показывает родителей, супруга и детей в одном блоке.

#### Александр II — два брака

```ged-person-full
@I44@
```

> Два брака в дереве родственников: @F11@ (Marie of Hesse-Darmstadt) + @F593@.

## Родственники

> Показывает **родственников** персоны: родители, братья/сёстры, супруг(и), дети.

**Когда использовать:** обзор ближайших родственников. Для визуальной диаграммы используйте `ged-diagram-relatives`.

**Синтаксис:** `@ID@ [@ID@ ...]`

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-relatives» (Ctrl+P)*

#### Николай II — семья с пятью детьми

```ged-relatives
@I37@
```

> Показывает родителей (Alexander III и Dagmar), супругу (Alexandra), и пятерых детей.

#### Александр II — два брака

```ged-relatives
@I44@
```

## Диаграммы

Все диаграммы интерактивны: **перетаскивание**, **зум**, кнопка **Expand to full width**, **Zoom in/out**, **Fit to view**.

### Параметр LVL (уровни поколений)

Опционально. Пишется **перед** `@ID@` на отдельной строке:

```
LVL:1    — 1 поколение
LVL:3    — 3 поколения
@I37@    — ID персоны
```

Если `LVL` не указан — берётся значение из настроек [[#Количество поколений по умолчанию]] (по умолчанию 3).

### ged-diagram-ancestors

> Диаграмма **предков** — дерево восходящих родственников (родители, бабушки/дедушки и т.д.).

**Синтаксис:** `LVL:N` (опционально) + `@ID@`

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-diagram-ancestors» (Ctrl+P)*

#### Предки Николая II

```ged-diagram-ancestors
LVL:5
@I37@
```

### ged-diagram-descendants

> Диаграмма **потомков** — дерево нисходящих родственников (дети, внуки и т.д.).

**Синтаксис:** `LVL:N` (опционально) + `@ID@`

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-diagram-descendants» (Ctrl+P)*

#### Потомки Александра III

С параметром `LVL:1`


#### Потомки Nicholas I

```ged-diagram-descendants

@I42@
```

### ged-diagram-hourglass

> **Песочные часы** — и предки, и потомки одновременно.

> ⚠️ **WARNING: производительность**
>
> Эта диаграмма строит **оба направления** (предки ↑ + потомки ↓). Для персон с большим количеством родственников рендеринг может занимать **значительное время** и потреблять много памяти.
>
> **Рекомендация:** всегда используйте `LVL:1` или `LVL:2` для безопасности. `LVL:3+` — только если вы уверены в размере дерева.

**Синтаксис:** `LVL:N` (опционально, рекомендуется 1-2) + `@ID@`

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-diagram-hourglass» (Ctrl+P)*

#### Николай II — песочные часы

```ged-diagram-hourglass

@I37@
```

#### Александр II — песочные часы

```ged-diagram-hourglass
LVL:2
@I44@
```

### ged-diagram-relatives

> ⚠️ **WARNING: производительность**
>
> Эта диаграмма строит **всех связанных родственников**. Для больших семей рендеринг может занимать **значительное время** и создавать очень большую диаграмму.

> **Рекомендация:** всегда используйте `LVL:1` или `LVL:2` для безопасности. `LVL:3+` — только если вы уверены в размере дерева.

**Синтаксис:** `LVL:N` (опционально, рекомендуется 1-2) + `@ID@`

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-diagram-relatives» (Ctrl+P)*

## Временные шкалы (ged-chronos)

### ged-chronos

> Интерактивная **временная шкала** на основе плагина [Chronos](https://github.com/clairefro/obsidian-plugin-chronos). Поддерживает директивы `gci:` (individual), `gcf:` (family), а также обычный Chronos DSL.

**Требования:** установлен плагин [Chronos Timeline](https://github.com/clairefro/obsidian-plugin-chronos).

**Синтаксис:**
- `gci: @ID@` — события персоны (все события из GEDCOM)
- `gcf: @ID@` — события семьи (брак/развод, события всех членов включая детей)
- `- [дата] описание` — произвольное событие
- `# комментарий` — комментарий
- Группировка: `{Group} Label | Place | Note`

Подробнее: [Chronos cheatsheet](https://github.com/clairefro/obsidian-plugin-chronos/blob/main/docs/chronos-cheatsheet.md)

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-chronos» (Ctrl+P)*

#### Николай II и Александра

```ged-chronos
gci: @I37@
gci: @I39@
```

#### Полная семья: родители + дети через семью

```ged-chronos
gci: @I37@
gci: @I39@
gcf: @F4@
```

> @F4@ — семья Nicholas II, включает всех пятерых детей.

#### Многопоколенная временная шкала

```ged-chronos
gci: @I42@
gci: @I44@
gci: @I40@
gci: @I37@
gcf: @F10@
gcf: @F9@
gcf: @F4@
```

> Nicholas I → Alexander II → Alexander III → Nicholas II — четыре поколения на одной шкале.

#### Произвольные события + GEDCOM

```ged-chronos
# Romanov Dynasty Timeline

gci: @I37@
gci: @I40@

- [1917-07-17] Russian Revolution
- [1918-07-18] Execution of the Romanovs
```

> Можно смешивать автоматические события из GEDCOM с произвольными событиями.

## Скрипты (ged-js)

### ged-js

> Выполняет **пользовательский JavaScript** с доступом к GEDCOM-данным через объект `ged`.

> ⚠️ **Внимание:** `ged-js` отключён по умолчанию. Включите в [[#Блоки ged-js]].

*Также пустой блок можно вставить через команду «GEDCOM: Вставить блок ged-js» (Ctrl+P)*

#### Контекст `ged`

| Поле | Описание |
|------|----------|
| `ged.service` | GedcomService — доступ ко всем данным GEDCOM |
| `ged.app` | Obsidian App — vault, workspace, metadataCache |
| `ged.container` | HTMLElement — контейнер для рендеринга |
| `ged.component` | MarkdownRenderChild — жизненный цикл |
| `ged.sourcePath` | Путь к текущему файлу заметки |

#### Методы рендеринга

| Метод | Описание | Пример |
|-------|----------|--------|
| `ged.el(tag, attrs, parent?)` | Создать DOM-элемент | `ged.el('div', { cls: 'my-class' })` |
| `ged.header(level, text)` | Заголовок h1-h6 | `ged.header(2, 'Title')` |
| `ged.paragraph(text)` | Параграф (plain text) | `ged.paragraph('Hello')` |
| `ged.span(text)` | Span (plain text) | `ged.span('text')` |
| `ged.renderMarkdown(md, el?)` | **Рендер markdown через Obsidian** — `**bold**`, `*italic*`, ссылки, таблицы | `await ged.renderMarkdown('**Bold**', ged.container)` |
| `ged.list(items[])` | Маркированный список | `ged.list(['a', 'b'])` |
| `ged.table(headers[], rows[][])` | HTML-таблица | `ged.table(['Name'], [['John']])` |
| `ged.formatDate(date)` | GEDCOM-дата → человекочитаемая | `ged.formatDate('1 JAN 1950')` → `'1 января 1950'` |

#### Основные методы GedcomService

**Персоны:**
- `ged.service.getIndividual('@ID@')` — получить персону по ID (`GedcomIndividual | null`)
- `ged.service.getAllIndividuals()` — массив всех персон

**Семьи:**
- `ged.service.getFamily('@ID@')` — получить семью по ID (`GedcomFamily | null`)
- `ged.service.getAllFamilies()` — массив всех семей

**Родственники:**
- `ged.service.getFamilyMembers('@ID@')` — объект:
  - `.father` / `.mother` — родители
  - `.siblings[]` — братья/сёстры
  - `.spouses[]` — супруги
  - `.children[]` — дети

#### Свойства GedcomIndividual

```javascript
{
    id,              // "@I42@"
    name,            // "Nicholas I Romanov"
    firstName,       // "Nicholas I"
    surname,         // "Romanov"
    birthDate,       // "1796" (сырой формат GEDCOM)
    birthPlace,      // "Tsarskoe Selo"
    deathDate,       // "1855"
    deathPlace,      // "St. Petersburg"
    sex,             // "M" или "F"
    familiesAsSpouse, // ["@F4@"] — семьи где персона супругом
    familiesAsChild,  // ["@F469@"] — семья где персона ребёнком
    events           // [{type, date, place}, ...] — все события из GEDCOM
}
```

#### Свойства GedcomFamily

```javascript
{
    id,              // "@F4@"
    husbandId,       // "@I37@"
    wifeId,          // "@I39@"
    childrenIds,     // ["@I46@", "@I47@", ...]
    marriageDate,    // "26 NOV 1894"
    marriagePlace,   // "Winter Palace"
    divorceDate,     // (может быть undefined)
    divorcePlace,
    events           // [{type: 'MARR', date, place}, ...] — все события семьи
}
```

#### Навигация — шаблоны использования

```javascript
// Получить персону
const person = ged.service.getIndividual('@I42@');

// Родители и сиблинги
const fm = ged.service.getFamilyMembers(person.id);
const father = fm.father;
const mother = fm.mother;
const siblings = fm.siblings;

// Супруг(а) и дети
const spouses = fm.spouses;
const children = fm.children;

// Семья где родился (родители)
const birthFamId = person.familiesAsChild?.[0];
const birthFam = ged.service.getFamily(birthFamId);

// Семья где был супругом
const spouseFamId = person.familiesAsSpouse?.[0];
const spouseFam = ged.service.getFamily(spouseFamId);

// События персоны
for (const event of person.events) {
    console.log(event.type, ged.formatDate(event.date), event.place);
}

// Форматирование дат
const birthDate = ged.formatDate(person.birthDate);  // "1 января 1895"
const deathDate = ged.formatDate(person.deathDate);  // "17 июля 1918"
const lifespan = `${birthDate} — ${deathDate}`;

// Markdown-рендеринг (bold, italic, ссылки, таблицы)
ged.renderMarkdown(`## ${person.name}
Родился **${birthDate}**, умер **${deathDate}**.`, ged.container);
```

#### Рассказ от лица Николая I о его родителях

```ged-js
const person = ged.service.getIndividual('@I42@');
const fm = ged.service.getFamilyMembers(person.id);

const pName = person.name;
const pBirth = ged.formatDate(person.birthDate);

ged.renderMarkdown(`## Мои родители

Я - **${pName}**, родился в **${pBirth}** году.

Мой отец — **${fm.father?.name || 'неизвестен'}** (${ged.formatDate(fm.father?.birthDate)}).
Мать — **${fm.mother?.name || 'неизвестна'}** (${ged.formatDate(fm.mother?.birthDate)}).`, ged.container);
```

#### Дети Николая II — таблица

```ged-js
const tsar = ged.service.getIndividual('@I37@');
const fm = ged.service.getFamilyMembers(tsar.id);

ged.header(2, `Дети ${tsar.name}`);

const headers = ['#', 'Имя', 'Дата рождения', 'Пол'];
const rows = [];

let i = 1;
for (const child of fm.children) {
    const sexLabel = child.sex === 'M' ? 'М' : 'Ж';
    rows.push([
        String(i++),
        child.name,
        ged.formatDate(child.birthDate) || '—',
        sexLabel
    ]);
}

ged.table(headers, rows);
ged.paragraph(`\nВсего детей: ${rows.length}`);
```

## Отладка

> **Совет:** Включите [[#Отладочные логи]] в настройках плагина и откройте **Вид → Переключить инструменты разработчика** (View → Toggle Developer Tools) — логи плагина выводятся во вкладку Console. Это полезно при поиске причин ошибок парсинга, проблем с диаграммами или некорректным рендерингом блоков.

## Устранение неполадок

### Блок не рендерится / показывает «Данные GEDCOM не загружены»

**Причина:** GEDCOM-файл не загружен.

**Решение:**
1. Откройте **Settings → GEDCOM Genealogy**
2. Проверьте **GEDCOM file path** — путь должен быть относительным корня vault
3. Убедитесь что `.ged` файл находится в vault (не за его пределами)
4. Перезагрузите Obsidian

### Диаграмма тормозит / зависает

**Причина:** слишком много персон для данного `LVL`.

**Решение:**
1. Уменьшите `LVL` до `1` или `2`:
   ```
   LVL:1
   @I37@
   ```
2. Особенно критично для [[#ged-diagram-hourglass]] и [[#ged-diagram-relatives]]
3. Уменьшите [[#Количество поколений по умолчанию]] в настройках

### ged-js блоки не работают

**Причина:** функция отключена в настройках (по умолчанию).

**Решение:**
1. Откройте **Settings → GEDCOM Genealogy**
2. Включите [[#Блоки ged-js]]
3. Перезагрузите Obsidian

### Ошибка «Плагин Chronos не найден» в ged-chronos

**Причина:** не установлен плагин Chronos Timeline.

**Решение:**
1. Установите плагин [Chronos](https://github.com/clairefro/obsidian-plugin-chronos) через Community plugins
2. Включите его в настройках
3. Перезагрузите Obsidian

### Файл не парсится / «Probably not a Gedcom file»

**Причина:** файл повреждён или в неправильной кодировке.

**Решение:**
1. Убедитесь что файл в формате GEDCOM 5.5 / 5.5.1
2. Проверьте кодировку — рекомендуется **UTF-8**
3. Попробуйте экспортировать заново из вашей генеалогической программы
